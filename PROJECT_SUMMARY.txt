# SaaS Multi-Tenant Billing & Document Platform — Project Summary

This document summarizes the development process and current status of the Multi-Tenant SaaS application.

## 1. Backend Architecture (Django REST Framework)
The backend is designed using a "Shared Database, Shared Schema" model, where every data row is linked to a specific `Tenant`.

### Key Features Implemented:
*   **Multi-Tenancy**: Central `Tenant` model to isolate company data.
*   **Custom Authentication**: Extended Django User model with `tenant_id` and `role` (Admin/Employee).
*   **JWT Security**: Secure login/signup using JSON Web Tokens (SimpleJWT).
*   **Secure Document Storage**:
    *   **Encryption**: Files are encrypted using AES-128 (Fernet) before being saved to the server's hard drive.
    *   **Decryption**: Files are automatically decrypted only when an authorized user downloads them.
    *   **Audit Logging**: Every upload/download is recorded in an activity timeline.
*   **Billing Engine**:
    *   Automated `Subscription` tracking.
    *   `Invoice` generation and status management.
    *   Mock Payment Gateway supporting UPI, Card, and Net Banking simulation.

### Important Backend Files:
*   `core/models.py`: Database definitions for Tenants, Users, Invoices, and Documents.
*   `core/views.py`: API logic for registration, login, dashboard stats, and file management.
*   `core/utils/encryption.py`: The logic that scrambles files for security.
*   `backend/settings.py`: Global configuration, including the secret encryption key.

---

## 2. Frontend Architecture (React + Vite)
The frontend is a modern, responsive Single Page Application (SPA).

### Key Components:
*   **Authentication Flow**: Login and Signup pages that interact with JWT tokens.
*   **Dynamic Dashboard**: A central workspace showing tenant-specific stats.
*   **Billing & Invoices**: A dedicated page to view payment history and simulate payments.
*   **Secure Document Manager**: 
    *   File upload interface.
    *   Encrypted file list with "Decrypt & Download" functionality.
*   **Global Layout**: Sidebar navigation and glassmorphism-inspired UI components.

### Important Frontend Files:
*   `src/App.jsx`: Main routing logic.
*   `src/pages/Dashboard.jsx`: Corporate workspace overview.
*   `src/pages/Documents.jsx`: Secure file manager.
*   `src/pages/Invoices.jsx`: Billing and payment simulator.
*   `src/index.css`: Custom premium styling system.

## 5. System Critics & Technical Review
*   **Database Scalability**: Currently using **SQLite**. Recommendation: Migrate to **PostgreSQL** for actual multi-tenant data isolation and performance.
*   **Asynchronous Processing**: File encryption is currently "synchronous" (stops the server while encrypting). Recommendation: Use **Celery + Redis** to handle large files in the background.
*   **Real-Time Engine**: The WebSocket layer is configured for **MemoryChannelLayer**. Production will require a **Redis** instance to handle multiple server workers.
*   **Payment Gateway**: Current implementation is a **Mock Simulation**. Real-world payment keys (Stripe/Razorpay) are required for actual money transactions.
